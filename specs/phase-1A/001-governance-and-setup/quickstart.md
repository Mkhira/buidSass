# Quickstart: Governance and Setup

**Branch**: `001-governance-and-setup` | **Date**: 2026-04-19

A contributor or agent implementing spec 001 works through these setup phases in order. Each section lists the deliverables and the verification step that proves them done.

---

## Phase A — Repository skeleton

**Deliverables**:
- Top-level folders per ADR-001 created (all empty except a `.gitkeep`).
- `.editorconfig` at root (tabs/spaces, line endings, charset per package type).
- `CLAUDE.md`, `.codex/system.md`, `GLM_CONTEXT.md` generated by `scripts/gen-agent-context.sh`.

**Verify**: `ls -1` at repo root shows exactly `apps/`, `services/`, `packages/`, `infra/`, `scripts/`, plus standard root files. Running `scripts/gen-agent-context.sh claude` exits 0 and produces `CLAUDE.md` with Principles 1–32 and the ADR Decisions table present.

---

## Phase B — Definition of Done

**Deliverables**:
- `docs/dod.md` with `## Universal Core` (8 items) and `## Applicability-Tagged Items` (5 triggers).

**Verify**: Open `docs/dod.md`. Every UC item in the data model is present. Every applicability trigger listed in the data model has a matching `### [trigger: ...]` section.

---

## Phase C — CI pipeline

**Deliverables**:
- `.github/workflows/lint-format.yml` — runs per-package formatters/linters on changed paths.
- `.github/workflows/contract-diff.yml` — emits OpenAPI artifact from backend build, runs oasdiff.
- `.github/workflows/verify-context-fingerprint.yml` — extracts fingerprint from PR description, recomputes, diffs.
- `.github/workflows/build-and-test.yml` — builds and tests all packages.

**Verify**: Open a draft PR with a deliberate formatting error. Confirm `lint-format` fails. Fix it, confirm it passes. Open a PR without `<!-- context-fingerprint: ... -->` in the description. Confirm `verify-context-fingerprint` fails.

---

## Phase D — Branch protection and CODEOWNERS

**Deliverables**:
- `.github/CODEOWNERS` with rules per the contract (constitution/ADR paths → 2 owners; catchall → 1).
- Branch protection on `main`: required checks, 1-reviewer default, signed commits, squash-only, no force push.
- Separate branch protection override for constitution/ADR paths (2 reviewers) — handled via CODEOWNERS.

**Verify**: Attempt to push directly to `main` — rejected. Open a PR without a code-owner review — merge button disabled. Modify the constitution file — confirm two approvals are required before merge is enabled.

---

## Phase E — PR template and scripts

**Deliverables**:
- `.github/pull_request_template.md` with spec link, Universal Core checklist, active-tag declaration field, summary.
- `scripts/gen-agent-context.sh` — generates all three agent context files + fingerprint.
- `scripts/extract-adr-block.sh` — extracts §7 from implementation plan for fingerprinting.
- `scripts/compute-fingerprint.sh` — computes the canonical SHA-256 hash for embedding.

**Verify**: Create a new PR — confirm the template appears pre-populated. Run `scripts/compute-fingerprint.sh` locally — output matches the hash the CI job computes on the same commit.
